## Funciones propias

4 pilares

1. Descomposición

2. Reconocimiento de patrones

3. Abstracción

4. Algoritmos

1. DIvidir para conquistar consiste en tomar un problema completo y romperlo
 en partes manejables.

2. Es observar tendencias o regularidades dentro de un problema si algo
 se repite o sigue una lógica constante, podemos automatizarlo.

3. Es filtrar la información innecesaria y quedarse solo con las características
que definen el problema. Es crear una representación simbolica de la realidad

4.Es el diseño de una serie de relgas ordenadas para resolver el problema.
Es el "plan de acción" que debe seguir el sistema.


***Tipos de interacción***

Es cuando curre un evento específico (clic) y el sistema responde con una acción única y aparecen
círculos. Es un interruptor de encendido y apagado,  o la acción y reacción.

En el código se suele usar dentro de la función mousePressed() o con un if(mousePressed)
